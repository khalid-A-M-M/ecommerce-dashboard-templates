# Ecommerce Dashboard Templates

This demo shows a compact version of a 5-day ecommerce dashboard automation sprint.

## What It Demonstrates

- Cleaning messy ecommerce orders.
- Normalizing product names.
- Removing cancelled, refunded, and zero-revenue orders.
- Adding profit calculations.
- Preparing a clean CSV for Power BI.
- Defining core DAX measures for revenue, profit, AOV, ROAS, and weekly growth.

## Files

| File | Purpose |
| --- | --- |
| `data/raw_orders.csv` | Messy dummy ecommerce orders |
| `data/clean_orders.csv` | Cleaned output generated by the script |
| `../02_Code_Library/clean_orders.py` | Reusable Python cleaning script |
| `power_bi_measures.dax` | Starter DAX measures |

## Run

From the `Sprint_Business` folder:

```bash
python 02_Code_Library/clean_orders.py --input 04_Portfolio_Demo/data/raw_orders.csv --output 04_Portfolio_Demo/data/clean_orders.csv
```

## Dashboard Pages

1. Overview: revenue, profit, orders, AOV, ROAS.
2. Products: top products by revenue and profit.
3. Ads: ad spend, ROAS, revenue by campaign source if available.

## Client Outcome

The client no longer has to combine weekly exports manually. Orders are cleaned into a dashboard-ready dataset, and the dashboard can refresh from a scheduled source.
